# "Программная реализация BFS, A* и Q-learning в задачах навигации интеллектуальных агентов в динамически изменяемых средах"
Руководство пользователя: запуск и анализ проекта
# 1. Системные требования
Операционная система: Windows / macOS / Linux

Python: версия 3.8 или выше

Оперативная память: от 512 МБ (рекомендуется 1 ГБ)

Свободное место на диске: около 50 МБ (для кода, библиотек и сохранения графиков)

2. Установка зависимостей
Перед первым запуском необходимо установить требуемые библиотеки. Для этого:

Откройте терминал (командную строку) в папке с проектом.

Выполните команду:

bash
pip install -r requirements.txt
Содержимое файла requirements.txt:

text
numpy
matplotlib
Если вы используете виртуальное окружение (рекомендуется), сначала создайте его и активируйте.

3. Структура проекта (основные файлы)
После распаковки архива вы увидите следующее:

text
проект/
├── maze.py                # генерация лабиринта, динамические изменения
├── bfs_solver.py          # поиск пути методом BFS
├── astar_solver.py        # поиск пути методом A*
├── q_learning_agent.py    # агент Q-learning (обучение, выбор действий)
├── visualization.py       # отрисовка лабиринта и графиков сходимости

└── requirements.txt       # список зависимостей
4. Запуск главного скрипта
Перейдите в папку с проектом и выполните:

bash
python main.py
Что происходит во время выполнения:

Автоматически запускается 10 независимых экспериментов (разные начальные лабиринты).

В каждом эксперименте:

Генерируется лабиринт 10×10 со случайными стенами (вероятность стены 0.3).

Агент Q-learning обучается в течение 500 эпизодов. Каждые 100 эпизодов лабиринт случайным образом изменяется (динамические препятствия).

BFS и A* запускаются на финальной версии лабиринта для получения эталонного пути.

По завершении всех запусков в консоль выводится статистика (средние длины путей и стандартные отклонения) для BFS, A* и Q-learning.

После сбора статистики показывается интерактивное окно с графиками для последнего (десятого) запуска:

Левый график: карта лабиринта, старт (зелёная точка), цель (красная точка), три пути:

BFS – зелёная сплошная линия,

A* – синяя пунктирная линия,

Q-learning – красная штрих-пунктирная линия.

Правый график: кривая сходимости Q-learning – по оси X номер успешного эпизода (в котором достигнута цель), по оси Y количество шагов. Горизонтальная зелёная линия – средний эталонный оптимум (BFS/A*).

6. Настройка параметров эксперимента (для продвинутых пользователей)
Вы можете изменить ключевые параметры в файле main.py (в функции main()):

Параметр	Значение по умолчанию	Описание
NUM_RUNS	10	Количество независимых запусков (чем больше, тем точнее статистика, но дольше выполнение)
MAZE_SIZE	10	Размер лабиринта (клеток по стороне)
WALL_PROB	0.3	Вероятность, что клетка – стена (0.2–0.4 – разумный диапазон)
EPISODES	500	Количество эпизодов обучения Q-learning
DYNAMIC_CHANGE_FREQ	100	Частота изменений лабиринта (через сколько эпизодов)
DYNAMIC_CHANGE_PROB	0.1	Вероятность переключения статуса каждой клетки при изменении
Пример: вы хотите провести 5 запусков на лабиринте 15×15 с изменениями каждые 50 эпизодов. Отредактируйте соответствующие строки в main.py.

7. Возможные ошибки и их решение
Ошибка	Вероятная причина	Решение
ModuleNotFoundError: No module named 'numpy'	Не установлены зависимости	Выполните pip install -r requirements.txt
ImportError: No module named 'matplotlib'	Аналогично	Установите matplotlib
Программа выполняется очень долго (> 5 минут)	Слишком много запусков или большой лабиринт	Уменьшите NUM_RUNS или EPISODES, MAZE_SIZE
График не появляется	Matplotlib выбран неинтерактивный бэкенд	Добавьте в начало visualization.py: import matplotlib; matplotlib.use('TkAgg')
Q-learning ни разу не достиг цели (steps_history пуст)	Слишком сложный лабиринт (нет пути)	Уменьшите WALL_PROB или увеличьте MAX_STEPS_PER_EPISODE в run_single_experiment
8. Сохранение результатов
Если вы хотите сохранить графики в файлы, добавьте в конец visualization.py после plt.show() строки:

Если вы готовите отчёт, сделайте скриншоты графиков и скопируйте итоговую статистику в текст.
